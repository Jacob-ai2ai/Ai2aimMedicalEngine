# Wireframes Directory

This directory contains wireframe specifications for all pages in the AI2AIM RX platform.

## Structure

Wireframes are organized by priority and numbered sequentially:

- `WIREFRAME_TEMPLATE.md` - Standard template for creating new wireframes
- `01-login.md` - Login page (HIGH priority)
- `02-dashboard.md` - Main dashboard (HIGH priority)
- `03-*.md` - Patient management pages (HIGH priority)
- `04-*.md` - Prescription pages (HIGH priority)
- `05-*.md` - Communication pages (HIGH priority)
- `06-*.md` - Appointment pages (HIGH priority)
- `07-*.md` - Sleep clinic pages (MEDIUM priority)
- `08-*.md` - Administrative pages (MEDIUM/LOW priority)

## Wireframe Status

See [WIREFRAME_INVENTORY.md](../WIREFRAME_INVENTORY.md) for complete status of all pages.

## Creating New Wireframes

1. Copy `WIREFRAME_TEMPLATE.md`
2. Fill in all sections
3. Follow the naming convention: `[number]-[page-name].md`
4. Update the inventory document

## Wireframe Format

All wireframes are in Markdown format with:
- ASCII art layouts
- Component breakdowns
- Interactive elements
- States & variations
- Data requirements
- User workflows
- Accessibility requirements
- Responsive breakpoints

## Priority Implementation

Wireframes are being created in priority order:
1. HIGH priority (13 pages) - Weeks 1-2
2. MEDIUM priority (17 pages) - Weeks 3-4
3. LOW priority (5 pages) - Week 5-6

See [WIREFRAME_PRIORITY_MATRIX.md](../WIREFRAME_PRIORITY_MATRIX.md) for detailed roadmap.
